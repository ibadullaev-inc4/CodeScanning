# Contributing Guide

## Обязательная настройка перед первым коммитом

Этот репозиторий использует **pre-commit hooks** для автоматической проверки кода. Это гарантирует, что все разработчики следуют одним стандартам.

### Быстрая настройка (требуется один раз)

```bash
# 1. Установить pre-commit
pip install pre-commit    # или: brew install pre-commit

# 2. Активировать hooks в репозитории
pre-commit install

# Готово! Теперь проверки будут запускаться при каждом коммите
```

### Что будет проверяться при коммите?

- ✅ Синтаксис YAML/JSON файлов
- ✅ Отсутствие пробелов в конце строк
- ✅ Отсутствие секретов (пароли, токены, ключи)
- ✅ Размер файлов (max 500KB)
- ✅ Конфликты при мерже
- ✅ Прямые коммиты в `main` ветку (запрещены)

### Полезные команды

```bash
# Запустить проверки вручную
pre-commit run --all-files

# Автоматически исправить ошибки (если возможно)
pre-commit run --all-files --fix

# Пропустить проверки (не рекомендуется!)
git commit --no-verify
```

## Workflow

1. Создай feature branch из `main`
2. Пиши код
3. Коммитяй — pre-commit сам проверит и исправит ошибки
4. Если что-то не прошло — исправь и коммитяй снова
5. Пушь и создавай PR
6. GitHub Actions автоматически запустит дополнительные проверки (CodeQL, Trivy)

## Если у тебя что-то не работает

```bash
# Переустановить hooks
pre-commit install --install-hooks

# Обновить все зависимости
pre-commit autoupdate

# Проверить статус
pre-commit run --all-files --verbose
```
